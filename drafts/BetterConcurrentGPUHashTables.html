Better Concurrent GPU hash tables

Date Fri 06 March 2020 Tags Programming

Reversible hash functions

Reversible hash functions: https://github.com/senderista/hashtable-benchmarks

Using these, we can store the hash in the table instead of the key. This helps Robin Hood, and resizing the table by realloc-and-grow (because that has to rehash each key), and backwards shift (because when shifting backwards you have to see if a key is at its ideal slot).

Bidirectional Linear Probing

https://github.com/senderista/hashtable-benchmarks Searches either forwards or backwards from the ideal slot. Uses a hash function that’s ordered, so key1<key2 always means hash(key1)<hash(key2), but what hash function is that (besides a translation)?
